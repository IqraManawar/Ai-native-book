# ROS 2 کی بنیادیں

ROS 2 (Robot Operating System 2) روبوٹکس ایپلیکیشنز بنانے کے لیے ایک اوپن سورس فریم ورک ہے۔

## ROS 2 کیا ہے؟

ROS 2 ایک مکمل آپریٹنگ سسٹم نہیں ہے بلکہ ایک مڈل ویئر فریم ورک ہے جو فراہم کرتا ہے:

- **کمیونیکیشن**: نوڈز کے درمیان پیغام رسانی
- **ہارڈویئر ابسٹریکشن**: سینسرز اور ایکچوایٹرز کے لیے معیاری انٹرفیس
- **ٹولز**: ویژولائزیشن، ڈیبگنگ، اور سمیولیشن
- **پیکجز**: تیار شدہ الگورتھم اور ڈرائیورز

## ROS 2 آرکیٹیکچر

### نوڈز

نوڈ ROS 2 کی بنیادی اکائی ہے - ایک عمل جو مخصوص کام انجام دیتا ہے۔

```python
import rclpy
from rclpy.node import Node

class MyNode(Node):
    def __init__(self):
        super().__init__('my_node')
        self.get_logger().info('نوڈ شروع ہو گیا!')

def main():
    rclpy.init()
    node = MyNode()
    rclpy.spin(node)
    rclpy.shutdown()
```

### ٹاپکس

ٹاپکس پبلش/سبسکرائب کمیونیکیشن کی اجازت دیتے ہیں:

```
پبلشر نوڈ → ٹاپک → سبسکرائبر نوڈ(ز)
```

### سروسز

سروسز درخواست/جواب کمیونیکیشن فراہم کرتی ہیں:

```
کلائنٹ → درخواست → سرور → جواب → کلائنٹ
```

### ایکشنز

ایکشنز طویل عرصے کے کاموں کے لیے ہیں جن میں فیڈ بیک ملتی ہے:

```
کلائنٹ → گول → سرور
        ← فیڈ بیک ←
        ← نتیجہ ←
```

## عام پیکجز

| پیکج | مقصد |
|------|-------|
| `nav2` | نیویگیشن اور پاتھ پلاننگ |
| `moveit2` | روبوٹ آرم موشن پلاننگ |
| `ros2_control` | ہارڈویئر کنٹرول انٹرفیس |
| `tf2` | کوآرڈینیٹ ٹرانسفارمز |

## لانچ فائلز

لانچ فائلز متعدد نوڈز کو ایک ساتھ شروع کرتی ہیں:

```python
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    return LaunchDescription([
        Node(
            package='my_package',
            executable='my_node',
            name='my_node'
        ),
    ])
```

## QoS پالیسیز

Quality of Service پالیسیز کمیونیکیشن کے رویے کو کنٹرول کرتی ہیں:

- **Reliability**: قابل اعتماد یا بہترین کوشش
- **Durability**: عارضی یا ٹرانزینٹ لوکل
- **History**: آخری یا تمام رکھیں

## باب کا خلاصہ

- ROS 2 روبوٹکس کے لیے مڈل ویئر فریم ورک ہے
- نوڈز، ٹاپکس، سروسز، اور ایکشنز کمیونیکیشن فراہم کرتے ہیں
- لانچ فائلز پیچیدہ سسٹمز کو منظم کرتی ہیں
- QoS پالیسیز کمیونیکیشن کو بہتر بناتی ہیں
