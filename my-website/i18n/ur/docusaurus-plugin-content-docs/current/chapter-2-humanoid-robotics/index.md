# ہیومنائیڈ روبوٹکس کی بنیادیں

اس باب میں، ہم ہیومنائیڈ روبوٹکس کی بنیادی باتوں کا جائزہ لیں گے - انسان نما روبوٹس جو انسانی جسم کی ساخت کی نقل کرتے ہیں۔

## ہیومنائیڈ روبوٹ کیا ہے؟

ہیومنائیڈ روبوٹ ایک روبوٹ ہے جو انسانی جسم کی طرح ڈیزائن کیا گیا ہے۔ اس میں عام طور پر شامل ہیں:

- **سر**: سینسرز اور کیمرے رکھتا ہے
- **دھڑ**: مرکزی پروسیسنگ اور توانائی کا ذخیرہ
- **بازو**: ہیرا پھیری اور تعامل کے لیے
- **ٹانگیں**: دو پیروں پر چلنے کی صلاحیت

### ہیومنائیڈ ڈیزائن کیوں؟

1. **انسانی ماحول**: ہماری دنیا انسانوں کے لیے ڈیزائن کی گئی ہے
2. **قدرتی تعامل**: لوگ انسان نما روبوٹس کے ساتھ زیادہ آرام سے بات چیت کرتے ہیں
3. **ورساٹائل**: مختلف کام انجام دے سکتے ہیں

## روبوٹ کائنیمیٹکس

کائنیمیٹکس حرکت کا مطالعہ ہے بغیر قوتوں پر غور کیے۔

### فارورڈ کائنیمیٹکس

جوائنٹ زاویوں سے آخری پوزیشن کا حساب:

```
اینڈ ایفیکٹر پوزیشن = f(θ₁, θ₂, ..., θₙ)
```

### انورس کائنیمیٹکس

مطلوبہ پوزیشن سے جوائنٹ زاویوں کا حساب:

```
(θ₁, θ₂, ..., θₙ) = f⁻¹(مطلوبہ پوزیشن)
```

## بیلنس اور چلنا

ہیومنائیڈ روبوٹس کے لیے بیلنس برقرار رکھنا اہم چیلنج ہے۔

### Zero Moment Point (ZMP)

ZMP وہ نقطہ ہے جہاں زمین کا رد عمل کوئی ٹارک پیدا نہیں کرتا۔ مستحکم چلنے کے لیے، ZMP سپورٹ پولیگون کے اندر ہونا چاہیے۔

### چلنے کا سائیکل

1. **سنگل سپورٹ**: ایک پاؤں زمین پر
2. **ڈبل سپورٹ**: دونوں پاؤں زمین پر
3. **سوئنگ فیز**: پاؤں آگے بڑھ رہا ہے

## کنٹرول آرکیٹیکچر

```
سینسرز → ادراک → منصوبہ بندی → کنٹرول → ایکچوایٹرز
```

### کنٹرول کی سطحیں

| سطح | کام | رفتار |
|-----|-----|-------|
| ہائی لیول | منصوبہ بندی | سست |
| مڈ لیول | رفتار کنٹرول | درمیانی |
| لو لیول | جوائنٹ کنٹرول | تیز |

## مشہور ہیومنائیڈ روبوٹس

- **Atlas** (Boston Dynamics): متحرک حرکت میں مہارت
- **Optimus** (Tesla): گھریلو اور صنعتی استعمال
- **ASIMO** (Honda): ابتدائی پیش رفت
- **Digit** (Agility Robotics): لاجسٹکس اور ڈیلیوری

## باب کا خلاصہ

- ہیومنائیڈ روبوٹس انسانی ساخت کی نقل کرتے ہیں
- کائنیمیٹکس جوائنٹ حرکت کو آخری پوزیشن سے جوڑتی ہے
- بیلنس ZMP کنٹرول سے حاصل ہوتا ہے
- کنٹرول آرکیٹیکچر متعدد سطحوں پر کام کرتا ہے
